# 04-本地启动手册.md

> 文档版本：V1.0 ｜ 日期：2026-04-30
> 适用：新人入职第 1 天 / 运维 / 测试
> 一句话目标：拷下代码 30 分钟内，能在本地把前后端跑起来，能在云托管发一个版本。

---

## 0. 结论先看

- 前端：Node 18 + pnpm 8 + 微信开发者工具，三条命令起。
- 后端：JDK 11 + Maven 3.6+，一条命令起。
- 部署：本地 `docker build` 出镜像 → 推到云托管 → 配 `/api/*` 路径 → 完。
- 验证用 `curl`，不要凭感觉判断"应该好了"。

---

## 1. 前置环境

| 软件 | 版本要求 | 校验命令 |
|---|---|---|
| Node.js | 18.x（LTS） | `node -v` |
| pnpm | 8.x（推荐）或 npm 9+ | `pnpm -v` |
| JDK | 11（云托管基线） | `java -version` |
| Maven | 3.6+ | `mvn -v` |
| Docker Desktop | 4.x（Windows/Mac） | `docker -v` |
| 微信开发者工具 | 稳定版 1.06+ | 手动看版本 |
| Git | 2.x | `git --version` |

> 注意：JDK 必须 11，不要 8 也不要 17。云托管默认镜像 `eclipse-temurin:11-jre-alpine`。

## 2. 前端启动

### 2.1 安装依赖

```bash
cd D:/code/运势/frontend
pnpm install
# 没有 pnpm 就用：npm install
```

### 2.2 开发模式（小程序）

```bash
pnpm dev:mp-weixin
# 产物输出到 frontend/dist/dev/mp-weixin/
```

### 2.3 开发模式（H5，仅本地联调）

```bash
pnpm dev:h5
# 默认 http://localhost:5173
```

### 2.4 用微信开发者工具打开

1. 微信开发者工具 → 「项目」→「导入项目」。
2. 项目目录选 `D:/code/运势/frontend/dist/dev/mp-weixin/`。
3. AppID 填团队 AppID。
4. 「详情」→「本地设置」勾选：
   - [x] 不校验合法域名、web-view 域名、TLS 版本以及 HTTPS 证书
5. 编辑 `frontend/src/config/env.ts`：
   - `USE_MOCK = false`（联调时）或 `true`（无后端时）。

### 2.5 生产构建

```bash
pnpm build:mp-weixin
# 用微信开发者工具上传 → 体验版 → 提审
```

### 2.6 常见问题

| 现象 | 处理 |
|---|---|
| `pnpm install` 卡死 | `pnpm config set registry https://registry.npmmirror.com` |
| `dist` 目录不更新 | 关掉微信开发者工具，删 `dist`，重新 `pnpm dev:mp-weixin` |
| 报 `Cannot find module '@/...'` | 检查 `tsconfig.json` paths 与 `vite.config.ts` alias 是否一致 |

## 3. 后端启动

### 3.1 拉依赖（首次较慢）

```bash
cd D:/code/运势/backend
mvn -DskipTests clean package
```

### 3.2 本地启动

```bash
mvn spring-boot:run
# 看到日志：Started FortuneApplication in 3.x seconds
# 默认端口 8080
```

### 3.3 验证（必做）

```bash
# 1. 健康检查
curl http://localhost:8080/api/stats/today

# 2. 模拟登录
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -H "X-WX-OPENID: mock_openid_local_dev" \
  -d '{"code":"local_mock_code"}'

# 3. 计算运势
curl -X POST http://localhost:8080/api/fortune/calc \
  -H "Content-Type: application/json" \
  -H "X-WX-OPENID: mock_openid_local_dev" \
  -d '{"birthday":"1995-08-15","birthHour":14,"gender":"M","tags":["career","love"],"calendarType":"solar"}'

# 4. 查历史
curl "http://localhost:8080/api/fortune/history?page=1&pageSize=10" \
  -H "X-WX-OPENID: mock_openid_local_dev"

# 5. 解锁深度
curl -X POST http://localhost:8080/api/fortune/unlock \
  -H "Content-Type: application/json" \
  -H "X-WX-OPENID: mock_openid_local_dev" \
  -d '{"recordId":"<填上一步返回的 recordId>","adToken":"mock_ad_token_xxx"}'
```

> 全部应该返回 `{"code":0,"message":"ok","data":{...},"traceId":"xxx"}`。

### 3.4 切换 profile

| profile | 用途 | 启动命令 |
|---|---|---|
| dev（默认） | 本地 + 云托管开发 | `mvn spring-boot:run` |
| prod | 云托管生产 | `java -jar app.jar --spring.profiles.active=prod` |

### 3.5 常见问题

| 现象 | 处理 |
|---|---|
| 端口 8080 被占用 | `mvn spring-boot:run -Dspring-boot.run.arguments=--server.port=8081` |
| Maven 拉包慢 | 配阿里云镜像，`~/.m2/settings.xml` 加 mirror |
| 启动报"mock json not found" | 检查 `src/main/resources/mock/` 三个 JSON 是否存在 |
| 时间差 8 小时 | 本地 JVM 加 `-Duser.timezone=Asia/Shanghai` |

## 4. Docker 本地构建

### 4.1 构建镜像

```bash
cd D:/code/运势/backend
mvn -DskipTests clean package
docker build -t fortune-backend:local .
```

### 4.2 本地运行

```bash
docker run --rm -p 8080:80 \
  -e SPRING_PROFILES_ACTIVE=dev \
  -e TZ=Asia/Shanghai \
  --name fortune-backend-local \
  fortune-backend:local
```

> 容器内监听 80（云托管约定），主机映射 8080。

### 4.3 进容器排查

```bash
docker exec -it fortune-backend-local sh
date                       # 看时区是不是 CST
ls /app                    # 看 jar 在不在
```

### 4.4 常见问题

| 现象 | 处理 |
|---|---|
| `COPY target/...: not found` | 没先 `mvn package` |
| 容器秒退 | `docker logs fortune-backend-local`，多半是端口或时区或 profile |
| Mac M1/M2 推上去云托管启动失败 | `docker build --platform linux/amd64 ...` |

## 5. 微信云托管发布

### 5.1 一次性准备

1. 微信公众平台 → 「云开发」→「云托管」→ 开通服务。
2. 创建环境：dev、prod 各一个，记下 envId。
3. 「服务列表」→「新建服务」→ 服务名 `fortune-backend`（必须与前端 `X-WX-SERVICE` 一致）。
4. 「访问设置」→ 公网访问 ON，路径配置新增 `/api/*`。
5. 配置「最小实例数 = 1」（生产，避免冷启动）；CPU 0.25 核 / 内存 0.5G 起步。

### 5.2 发布方式 A：上传镜像（推荐）

```bash
cd D:/code/运势/backend
mvn -DskipTests clean package
docker build --platform linux/amd64 -t fortune-backend:v1.0.0 .

# 登录云托管镜像仓库
docker login <region>.tcr.tencentcloudcr.com -u <账号> -p <临时密码>

docker tag fortune-backend:v1.0.0 <region>.tcr.tencentcloudcr.com/<ns>/fortune-backend:v1.0.0
docker push <region>.tcr.tencentcloudcr.com/<ns>/fortune-backend:v1.0.0

# 云托管控制台 → 版本管理 → 新建版本 → 选刚 push 的镜像 → 发布
```

### 5.3 发布方式 B：绑定 Git 仓库

1. 云托管控制台 → 服务 → 「版本管理」→「新建版本」。
2. 构建方式选「代码仓库」→ 授权并选择 backend 仓库 + 分支。
3. Dockerfile 路径填 `Dockerfile`。
4. 提交后云托管自动构建并部署。

### 5.4 前端切到云托管

```ts
// frontend/src/config/env.ts
export const USE_MOCK = false
// 确认 cloudEnvId / service 与小程序后台一致
```

重新 `pnpm build:mp-weixin` → 微信开发者工具上传 → 体验版扫码验证。

### 5.5 上线后线上验证

```js
// 在微信开发者工具 console 直接调
wx.cloud.callContainer({
  config: { env: 'fortune-prod-1xxxxxxx' },
  path: '/api/stats/today',
  method: 'GET',
  header: { 'X-WX-SERVICE': 'fortune-backend' },
  success: console.log, fail: console.error
})
```

> 线上风险：
> 1. 首次上线务必灰度 10% → 50% → 100%。
> 2. 旧版本至少保留 1 个，回滚靠"切流量"，1 分钟内可回。
> 3. 上线后 5 分钟内盯日志 + 监控（CPU、内存、5xx 率）。
> 4. 不要在生产 envId 上做 `delete /api/fortune/history`（清空全部）测试。

## 6. 全链路验证命令速查

| 阶段 | 命令 | 期望 |
|---|---|---|
| 后端本地启动 | `curl http://localhost:8080/api/stats/today` | code:0 |
| Docker 本地 | `docker run -p 8080:80 fortune-backend:local` + curl | code:0 |
| 云托管开发环境 | 微信开发者工具 console 调 callContainer | code:0 |
| 生产 | 体验版扫码走主流程 | UI 完整、无 5xx |

## 7. 一句话备忘

- 改后端 → `mvn spring-boot:run` → curl 验证 → 提交 → 云托管发版。
- 改前端 → `pnpm dev:mp-weixin` → 微信开发者工具刷新 → 真机预览。
- 出问题 → 先看 traceId，再看后端日志，再看云托管日志，最后再怀疑前端。
