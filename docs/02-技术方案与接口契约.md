# 技术方案与接口契约 v1.0

> 用途：前后端联调对齐基线 / 微信云托管部署依据 / 后续接入真实数据库的演进基线
> 适用范围：第 1 期（Mock 数据期），无 DB、无第三方支付、流量主广告变现

---

## 0. 结论先行

- 前端：**uni-app + Vue3 + TypeScript + uni-ui**，编译目标以微信小程序为主。
- 后端：**Spring Boot 2.7.18**（云托管 JDK8/11 兼容性最稳），统一响应 + 全局异常 + 内存 Mock。
- 第 1 期不连 DB/Redis，所有数据来自 `resources/mock/*.json` 加载到内存 Map。
- 调用层封装 `wx.cloud.callContainer`。
- 运势算法：**(openid+生日+性别+yyyyMMdd) → MD5 → 取模 → 1~100**，同一人当天同结果。
- 文案严禁封建迷信、灾祸、命理决定论。

## 1. 技术选型

### 1.1 前端

| 选型 | 理由 |
|---|---|
| uni-app | 一套代码覆盖微信小程序 + H5 + 后续 App |
| Vue3 (Composition API) | 与 uni-app 3.x 主线一致 |
| TypeScript | 强约束 API 契约，编译期暴露字段错配 |
| uni-ui | 官方组件库，包体小 |
| Pinia | Vue3 官方推荐，比 Vuex 更轻 |

### 1.2 后端

| 选型 | 理由 |
|---|---|
| Spring Boot 2.7.18 | 云托管默认镜像兼容 JDK8/11，长期维护 |
| Lombok | 减少 DTO/VO 样板代码 |
| Hutool 5.8.x | MD5、随机、日期、JSON 一把梭 |
| Jackson | Spring Boot 默认序列化器 |

### 1.3 假数据方案

**采用：静态 JSON 文件 + 启动时加载到内存 Map**

文案池放 `resources/mock/`：
- `fortune_text.json`：按 `level → dimension → text[]` 组织
- `lucky_pool.json`：颜色/数字/方位池
- `yi_ji_pool.json`：宜忌池

启动时 `@PostConstruct` 一次性加载。

## 4. 核心 API 契约

### 4.0 通用约定

**统一响应**：

```json
{ "code": 0, "message": "ok", "data": {}, "traceId": "f3a1c9e2" }
```

**code 约定**：
- `0` 成功
- `1xxx` 参数错误（1001 缺失、1002 格式、1003 范围）
- `2xxx` 业务错误（2001 未登录、2002 记录不存在、2003 未解锁）
- `4xxx` 限流（4001）
- `5xxx` 服务端异常

**时间格式**：ISO8601 字符串 `2026-04-30T10:00:00+08:00`，日期 `2026-04-30`。

## 5. 统一响应 / 异常 / 跨域 / 限流

- **统一响应**：`ApiResponse.success(data)` / `ApiResponse.fail(code, msg)`，禁止裸返实体。
- **全局异常**：`BizException` → 业务码；`MethodArgumentNotValidException` → 1002；`Exception` → 5000，不回传堆栈。
- **TraceId**：`TraceIdFilter` 入口生成或读 Header，放入 MDC + 响应体。
- **CORS**：`callContainer` 走云托管内网无需 CORS；H5 调试期放行 `*`。
- **限流**：内存令牌桶。`/calc` 单 openid 1 分钟 10 次；`/unlock` 单 openid 1 分钟 5 次；触发返回 4001。

## 6. 运势算法

```
seed = MD5(openid + birthday + gender + yyyyMMdd(today))
score = (seed_前4字节转int 的绝对值) % 100 + 1   // 1~100
```

- 同一 openid + 同一生日 + 同一天 → 同一 score。
- 维度评分：基于同一 seed 不同字节段。

**等级映射**：

| score | level | 文案池 |
|---|---|---|
| 90-100 | 大吉 | level_excellent |
| 70-89 | 吉 | level_good |
| 40-69 | 平 | level_normal |
| <40 | 凶 | 本期池子留空，下沉到"平" |

**文案匹配**：`pool[level][dimension]` 中按 `seed % len` 取一条，每桶至少 20 条。

**幸运三件套**：12 色、9 数、8 方位，按 seed 取。

**宜忌**：宜池 30、忌池 30，宜抽 3-5、忌抽 1-3，保证宜多于忌。

**禁用词**（命中即拒上线）：命运、注定、天命、克、煞、犯太岁、化解、消灾、改运、转运符、破财、横祸、血光、姻缘已定、必然、一定会、不可避免

**底部固定**：`tips` = "本结果基于趣味算法生成，仅供娱乐参考，不构成任何医疗、投资、婚恋等决策建议。"

## 7. 部署

### 7.1 Dockerfile

```dockerfile
FROM eclipse-temurin:11-jre-alpine
WORKDIR /app
COPY target/fortune-backend.jar app.jar
ENV SERVER_PORT=80
ENV SPRING_PROFILES_ACTIVE=prod
ENV TZ=Asia/Shanghai
EXPOSE 80
ENTRYPOINT ["sh","-c","java -Xms256m -Xmx512m -jar app.jar"]
```

要点：EXPOSE 80（云托管默认）；时区必设 Asia/Shanghai，否则跨天异常。

### 7.2 环境变量

| 变量 | 说明 | 默认 |
|---|---|---|
| SERVER_PORT | HTTP 端口 | 80 |
| SPRING_PROFILES_ACTIVE | profile | prod |
| TZ | 时区 | Asia/Shanghai |
| RATE_LIMIT_CALC_PER_MIN | calc 限流 | 10 |

### 7.3 callContainer 注意事项

- 走云托管内网，无需 ICP 备案，无需 CORS。
- 小程序后台配置 **服务路径白名单**：`/api/*`。
- Header 中 `X-WX-OPENID`、`X-WX-APPID` 由云托管自动注入。
- `Content-Type: application/json` 必显式设置。

## 10. 风险与回滚

| 风险 | 概率 | 影响 | 应对 | 回滚 |

| 文案踩"封建迷信"被下架 | 中 | 高 | 禁用词扫描 + 双签 | 紧急下线 + 维护中 |

| 云托管冷启动慢 | 中 | 中 | 最小实例 ≥1 | 前端 loading + 重试 |
| 跨天时区 bug | 低 | 中 | TZ + 单测 | 修 TZ 重发 |
