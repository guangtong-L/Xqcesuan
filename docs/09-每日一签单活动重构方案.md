# 09-每日一签单活动重构方案

> 日期：2026-05-10  
> 状态：已进入实现  
> 适用角色：产品 / 前端 / 后端 / 测试 / 运营

## 1. 结论

项目从“多功能运势工具”收敛为“每日一签单活动”。用户可见主链路只保留：

```text
首页抽签入口 → 今日签结果 → 本地抽签记录 → 我的设置
```

本次不涉及订单、支付、库存、会员、优惠券、退款、结算，风险主要来自内容合规、广告频控和旧入口残留。

## 2. 产品边界

### 保留

- 每日一签活动首页
- 今日签抽取
- 签文、释义、宜忌展示
- 本地抽签记录，最多 30 条
- 我的页：登录展示、每日提醒、清缓存、协议入口、反馈入口
- Banner / 历史信息流广告位
- 娱乐参考免责声明

### 下线用户入口

- 今日运势测算
- 信息录入
- 运势结果页
- 八字流年
- 星座匹配

说明：旧页面文件暂不物理删除，避免一次性破坏编译与后续回滚；用户可见路由已从 `pages.json` 移除。

## 3. 接口边界

### 当前主接口

```http
GET /api/fortune/sign
```

返回：

- `signNo`
- `signName`
- `poem`
- `explain`
- `yi`
- `ji`
- `date`
- `tips`

### 暂不作为主链路

- `/api/fortune/calc`
- `/api/fortune/bazi`
- `/api/fortune/match`
- `/api/fortune/unlock`
- `/api/fortune/history`

## 4. 验收标准

### 首页

- Given 用户打开小程序
- When 首屏加载完成
- Then 只看到每日一签活动，不出现八字、星座匹配、今日运势测算入口

### 今日签

- Given 用户点击“抽取今日签”
- When 抽签接口返回成功
- Then 展示签号、签名、四句签诗、释义、宜忌和免责声明

### 抽签记录

- Given 用户已完成一次抽签
- When 进入记录页
- Then 能看到该次抽签摘要，并支持查看、删除、清空

### 合规

- Given 任一用户可见页面
- When 浏览签文与说明
- Then 文案必须保持“娱乐参考”，不得出现命运承诺、灾祸预测、付费占卜或导流外站

## 5. 风险等级

- 风险等级：P2
- 原因：不涉及资金链路，但涉及内容合规、广告体验和旧功能下线后的入口一致性。

## 6. 测试建议

- H5 路由冒烟：首页 → 今日签 → 记录 → 我的
- 微信小程序真机：静默登录、抽签接口、广告位、订阅提醒入口
- 回归检查：TabBar 不再出现旧功能入口
- 合规扫描：签文 JSON 与页面静态文案不包含禁用词
