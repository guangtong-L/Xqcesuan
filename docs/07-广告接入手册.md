## 2. 运营操作（一次性，约 30 分钟）

### 2.1 开通流量主

**前置门槛（满足任一即可）**：
- 个人小程序：连续 7 天访问用户 ≥ 1000
- 企业小程序：直接申请，无门槛
- 小游戏：累计独立访客 ≥ 1000

**操作路径**：
```
微信公众平台（mp.weixin.qq.com）
→ 推广
→ 流量主
→ 开通流量主
→ 完成主体验证（企业需上传执照、对公账户等）
```

提交后通常 1-3 个工作日审核通过。审核期间可以先用代码里的 dev 占位 unitId，**不影响开发**。

### 2.2 申请 6 个广告位

开通后回到「流量主 → 广告位管理 → 新建广告位」。按下表逐一申请：

| 序号 | 广告位名称（建议命名） | 类型 | 备注 |
|---|---|---|---|
| 1 | fortune-home-banner | 原生模板 | 首页底部 |
| 2 | fortune-result-native | 原生模板 | 结果页 |
| 3 | fortune-result-reward | 激励式 | 结果页激励视频 |
| 4 | fortune-result-interstitial | 插屏式 | 结果页离开 |
| 5 | fortune-history-native | 原生模板 | 历史列表 |
| 6 | fortune-mine-banner | 原生模板 | 个人中心 |

> **注意**：
> 1. 微信已合并 Banner 与原生模板为同一类型「原生模板」，新建即选它。
> 2. 激励视频每个小程序最多创建 5 个广告位（5 个 unitId 共享后端配额，单 unitId 独立计费）。
> 3. 插屏每个小程序最多创建 5 个。
> 4. 创建后立刻拿到 unitId，格式形如 `adunit-1a2b3c4d5e6f7g8h`。

### 2.3 把 6 个 unitId 给前端

直接编辑 `frontend/src/config/ads.ts`，替换 PROD 区：

```ts
const PROD = {
  HOME_BANNER:         'adunit-1a2b3c4d5e6f7g8h',  // ← 真 ID
  RESULT_NATIVE:       'adunit-2b3c4d5e6f7g8h9i',
  RESULT_REWARD:       'adunit-3c4d5e6f7g8h9i0j',
  RESULT_INTERSTITIAL: 'adunit-4d5e6f7g8h9i0j1k',
  HISTORY_NATIVE:      'adunit-5e6f7g8h9i0j1k2l',
  MINE_BANNER:         'adunit-6f7g8h9i0j1k2l3m'
}
```

**只改这 6 行**，其他代码零改动。

### 2.4 激励视频回调验签（P2 阶段，本期可跳过）

本期 `unlock` 接口的 `adToken` 是 mock，不验签。线上为了防作弊（有人模拟 unlock 请求白嫖深度内容），需要：

1. 流量主后台 → 激励广告 → 服务端回调 → 配置回调 URL：
   ```
   https://<云托管域名>/api/fortune/ad-callback
   ```
2. 微信会在用户看完广告后回调你的接口，带签名 `sign`、`trans_id`、`user_openid` 等。
3. 后端按文档 [https://developers.weixin.qq.com/miniprogram/dev/component/ad-rewarded-video.html](激励视频回调) 验签，验签通过才允许该用户后续 unlock。

> **本期决策**：MVP 灰度期不接，DAU > 1 万再接，避免过早优化。

## 3. 代码侧使用方式

### 3.1 Banner / 原生模板

```vue
<script setup lang="ts">
import AdBanner from '@/components/AdBanner/index.vue'
import { ADS } from '@/config/ads'
</script>

<template>
  <AdBanner :unit-id="ADS.HOME_BANNER" />
</template>
```

**特性**：
- 仅微信小程序生效（H5/App 自动隐藏）
- 加载失败自动折叠（不撑空白）
- 默认 30s 轮播一次（可传 `:ad-intervals="60"` 调整）

### 3.2 激励视频

```ts
import { showRewardedVideo } from '@/utils/ad'
import { ADS, AD_LIMIT } from '@/config/ads'

async function onUnlockClick() {
  if (rewardCount.value >= AD_LIMIT.REWARD_PER_PAGE) {
    uni.showToast({ title: '今天看得有点多啦', icon: 'none' })
    return
  }
  const ok = await showRewardedVideo(ADS.RESULT_REWARD)
  if (ok) {
    // 完整观看，发奖励 / 调 unlock 接口
    await unlockApi()
  } else {
    uni.showToast({ title: '完整观看才能解锁哦', icon: 'none' })
  }
}
```

**返回值**：
- `true` = 用户完整观看完成
- `false` = 中途关闭、加载失败、非小程序环境

### 3.3 插屏

```ts
import { showInterstitial } from '@/utils/ad'
import { ADS, AD_LIMIT } from '@/config/ads'

onUnmounted(() => {
  showInterstitial(ADS.RESULT_INTERSTITIAL, AD_LIMIT.INTERSTITIAL_PER_DAY)
})
```

**自带特性**：
- 启动 5 秒内静默（避免冷启动打扰，被微信判"诱导"）
- 单用户单日次数本地频控（默认 1 次，可在 `AD_LIMIT.INTERSTITIAL_PER_DAY` 改）
- 拉取失败静默忽略

## 4. 频控配置

`frontend/src/config/ads.ts` 顶部：

```ts
export const AD_LIMIT = {
  REWARD_PER_PAGE: 4,             // 单页激励视频次数
  INTERSTITIAL_PER_DAY: 1,        // 单用户单日插屏次数
  INTERSTITIAL_COLD_GUARD_SEC: 5  // 启动后多少秒内不允许出插屏
}
```

调整这 3 个数字 → 全局生效，无需翻代码。

## 6. 数据查看

### 6.1 实时收入
```
小程序后台 → 推广 → 流量主 → 数据中心
- 当日预估收入
- 各广告位曝光、点击、eCPM
- 7 / 30 日趋势
```

### 6.2 自定义事件交叉
```
小程序后台 → 数据分析 → 自定义分析
- ad_reward_show / complete / close / fail
- ad_interstitial_show / fail
按事件聚合，可算激励视频完成率
```

### 6.3 收入预估口径

```
日广告收入 = Σ(广告位 eCPM × 曝光数 / 1000)

参考 eCPM（业内中位）:
  原生模板/Banner ≈ ¥ 3 ~ 8
  插屏           ≈ ¥ 8 ~ 20
  激励视频       ≈ ¥ 30 ~ 80

DAU = 5000 单日预估 ≈ ¥ 100 ~ 200
ARPU ≈ ¥ 0.02 ~ 0.04
```

详见 `05-埋点与广告位.md` 第 4 节。

## 7. 常见问题

| 现象 | 原因 | 解决 |
|---|---|---|
| 真机进结果页一片空白 | unitId 还是 `adunit-xxxxxxxxxxxx` 占位 | 把 `ads.ts` 的 PROD 区改成真 ID |
| 真机激励视频弹"adUnitId 不正确" | unitId 写错 / 流量主未审核通过 | 后台核对 ID，等审核 |
| 真机插屏从不弹 | 启动 5s 内进结果页就退出了 | 等 5s 后再进 |
| 第 2 次进结果页插屏不弹 | 已触发当日频控（设计如此） | 改 `AD_LIMIT.INTERSTITIAL_PER_DAY=2` 试 |
| 激励视频看完没解锁 | 后端 `/api/fortune/unlock` 接口报错 | 看后端日志 + traceId 排查 |
| 收入数据为 0 | 提审包还没审核通过；只有正式版/体验版才有真实曝光 | 提审上线后等 T+1 |
| 流量主审核被驳回 | 类目选错（选了"星座命理"被风控） | 改回"工具-实用查询" |
