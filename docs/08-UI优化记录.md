## 二、设计 Token 体系（核心）

### 1. 颜色

| 类别 | Token | 值 | 用途 |
|---|---|---|---|
| 主色 | `$color-primary` | `#7c5cff` | 默认强调（保留产品确认） |
| 主色浅 | `$color-primary-light` | `#ede8ff` | chip/tag 底 |
| 主色软 | `$color-primary-soft` | `#b39dff` | 渐变末端 |
| 爱情/匹配 | `$color-love` | `#ff5da6` | 匹配页评分、上上签 |
| 健康/宜 | `$color-health` | `#51c878` | 维度卡-健康、宜 |
| 财运/警示 | `$color-wealth` | `#f0ad4e` | 维度卡-财运 |
| 事业/冬季 | `$color-career` | `#4a90e2` | 维度卡-事业 |
| 错误/忌 | `$color-danger` | `#ff6b6b` | 比原 `#ff5050` 更柔和 |
| 季节 4 色 | `$color-spring/summer/autumn/winter` | 绿/橙/金/蓝 | 八字流年 4 季度 |
| 文字 5 档 | `$color-text/strong/body/sub/mute` | `#1f1f24 → #aaaaaa` | 替代原 6 档随机用色 |
| 背景 | `$color-bg / card / divider` | 灰白系 | 统一卡片底/分割线 |

### 2. 字号阶梯（rpx）

`xs:22 / sm:24 / md:26 / base:28 / lg:30 / xl:32 / 2xl:36 / 3xl:48 / display:96`

> 替代原本散乱的 22/24/26/28/30/32/36/40/56/64/96 共 11 档无序使用。

### 3. 圆角阶梯

`sm:12 / md:16 / lg:24 / xl:32 / pill:999rpx`

### 4. 间距阶梯

`1:4 / 2:8 / 3:16 / 4:24 / 5:32 / 6:48`（与字号、圆角同节奏）

### 5. 阴影 3 档

| Token | 用途 |
|---|---|
| `$shadow-sm` | 普通卡片 `0 2rpx 8rpx rgba(31,31,36,.04)` |
| `$shadow-md` | hero / CTA `0 8rpx 24rpx rgba(124,92,255,.12)` |
| `$shadow-lg` | 浮层 `0 12rpx 40rpx rgba(124,92,255,.18)` |

### 6. 复用 Mixin

| Mixin | 用途 |
|---|---|
| `card-base` | 默认卡片（白底 + 大圆角 + 轻阴影 + 默认内边距） |
| `card-hero` | Hero 大圆角 + 中阴影 |
| `btn-primary` | 主按钮（紫渐变 + 胶囊 + disabled 态） |
| `btn-secondary` | 次按钮（白底 + 主色文字） |
| `title-h1/h2/h3 / title-section` | 标题层级 |
| `disclaimer` | 免责声明统一样式（页面底部） |
| `hero-gradient-purple` | 紫渐变 hero |
| `hero-soft-pink` | 粉色柔和卡片（mine 页头部用） |
| `row-flex` | label + value 横排 |

## 三、改前 → 改后 关键对比

### 1. CSS 变量利用率：0% → 100%

- **改前**：`App.vue` 定义了 `--color-primary` 等 7 个变量，**8 个页面 0 引用**，全部硬编码 `#7c5cff` 等
- **改后**：所有页面通过 `@use '@/styles/tokens.scss' as *;` 引入 SCSS Token；`App.vue` 同步挂全套 CSS 变量副本（运行时切换主题用）

### 2. 紫色渐变重复 3 次 → 收敛为 1 个 Mixin + 差异化

| 页面 | 改前 | 改后 |
|---|---|---|
| `index/hero` | `linear-gradient(135deg, #7c5cff 0%, #b39dff 100%)` | `@include hero-gradient-purple;` + 装饰星点 |
| `bazi/hero` | 同上 | `@include hero-gradient-purple;`（保留紫色，但内容差异化） |
| `mine/header` | 同上（视觉雷同） | `@include hero-soft-pink;` 柔粉卡片 |

### 3. 卡片样式：4 种写法 → 1 个 Mixin

| 改前 | 改后 |
|---|---|
| `result__yiji-col`：圆角 24 / padding 28 | `@include card-base` |
| `result__lucky`：圆角 24 / padding 32 | `@include card-base` + 内部 `padding: $space-5` |
| `mine__signin`：圆角 24 / padding 28 | `@include card-base` |
| `bazi__card`：圆角 24 / padding 28 | `@include card-base` |
| `history__item`：圆角 24 / padding 28 / **无阴影** | `@include card-base` 衍生 + 左侧 level 色条 |

### 4. 按钮：3 种风格 → 1 个 Mixin

| 改前 | 改后 |
|---|---|
| `calc__submit`：纯紫 + 圆胶囊 | `@include btn-primary`（紫渐变 + 阴影 + disabled 态） |
| `match__btn`：纯紫 + 圆胶囊 | `@include btn-primary` |
| `bazi__retry`：纯紫 + 圆胶囊 | `@include btn-primary` |
| `result__retry-btn`：纯紫 + 圆胶囊 | `@include btn-primary` |
| `history__empty-btn`：纯紫 + 胶囊 | `@include btn-primary` |
| `sign__btn`：暖棕 `#c97b3f`（古风专属） | 保留古风 + 升级为渐变 + 阴影（不走通用 mixin） |

### 5. 文字色：6 档随机 → 5 档梯度

`#1f1f24 / #333 / #555 / #666 / #888 / #aaa` → `text(1f1f24) / strong(333) / body(555) / sub(888) / mute(aaa)`，删除 `#666` 这种"中间值"。

### 6. 历史记录：缺少 level 视觉提示 → 左色条按等级着色

```html
<view :class="`history__item--${levelClass(item.level)}`">
```

- 大吉/上上签 → 粉色 `$color-love`
- 吉/上签/小吉 → 紫色 `$color-primary`
- 平/凶 → 灰色 `$color-text-mute`

### 7. 八字流年：4 季度无差异 → 春绿/夏橙/秋金/冬蓝

通过 `seasonOf(q)` 函数解析 `q.key/q.name`（兼容 `春/spring/q1` 等多种格式），输出 `bazi__quarter--spring/summer/autumn/winter` 修饰类。每张卡：
- 顶部 6rpx 色条
- 内部从季节色 6% 透明度 → 白底 的浅渐变

### 8. 匹配页：单字符 ❤ → 渐变连接器 + 💞

- **改前**：`<text class="match__heart">❤</text>` 字号 48rpx，单字符在双 picker 中间显得突兀
- **改后**：垂直 `线 + 💞 + 线`，线为粉→紫渐变（`$color-love → $color-primary`），更有"两边都很重要 + 关系连接"的语义

### 9. 签页：签筒过大 → 88rpx + 仿古签纸升级

- 签筒 emoji `120rpx → 88rpx`，整体更精致
- 签筒底色从纯白 → 米黄渐变 `#fff5e6 → #fce5c4`
- 签纸边框升级：单层 `#e8d4b0` → 双层「烫金外环 + 内白圈 + 烫金内框」（`box-shadow: inset` 实现）
- 楷体字距 `4rpx → 6rpx`，行高 1.7

### 10. ScoreRing：静态 → 缓动入场 + 渐变

- 入场动画：分数从 0 → 目标值，0.6s easeOutCubic，治愈感强
- conic-gradient 加中间渐变停靠点（浅紫 → 主紫），更有层次
- 新增 `level` 可选 prop，传入则在分数下显示 chip

### 11. DimensionCard：无差异 → 按维度着色 + emoji

- 4 个维度的左色条 + 分数颜色按 key 自动着色：
  - 爱情 `💗` → `$color-love`
  - 事业 `💼` → `$color-career`
  - 财运 `💰` → `$color-wealth`
  - 健康 `🌿` → `$color-health`
- 蒙层胶囊文字加阴影 + 加大 padding，视觉更"可点"

### 12. 文字对比度修正

- `history__item-meta` 从 `#aaa` → `$color-text-sub` (`#888`)
  在白底上 #aaa 对比度仅 ≈ 2.85，#888 ≈ 4.6（满足 WCAG AA 4.5）
- 副文案 `disclaimer` 保留 `#aaa`（仅在大段空白上使用，且字号 22rpx 仅装饰性）

## 七、上线建议（线上风险提示）

1. **建议灰度发布**：本次为大范围视觉改动，虽业务逻辑零变动，但主色渐变 / 卡片阴影会改变截图效果，建议先 10% 灰度观察 1-2 天
2. **观察广告填充率**：`AdBanner` 的 margin 从 `16rpx` 略调，但容器尺寸不变，理论不影响 SDK 渲染；上线后需观察小程序后台的"广告填充率 / 收益"是否波动
3. **观察用户停留**：历史页加了 `-webkit-line-clamp: 2`，长摘要会被截断，若反馈"看不到全文"需改回不截断
4. **微信小程序 CSS 变量兼容性**：基础库 ≥ 2.7.1 才支持，建议在 `manifest.json` 设置最低基础库 `2.10.0`（如未设置）
5. **回滚方案**：所有改动集中在 `<style>` 段 + 1 个新增 `tokens.scss`，回滚时 `git revert` 单次提交即可
