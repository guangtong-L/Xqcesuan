> 文档版本：V1.0 ｜ 日期：2026-04-30 ｜ 状态：基线

## 0. 结论先看

- 生产链路：**uni-app → wx.cloud.callContainer → 微信云托管 → Spring Boot**，无需 CORS、无需备案。
- 本地链路：**uni-app H5 → uni.request → http://localhost:8080**，前端用 `USE_MOCK` 开关切换"静态 JSON / 真实接口"。
- openid 永远不前端传，由云托管自动注入 `X-WX-OPENID`，本地调试用 mock 值伪造。
- 所有错误码 → toast 文案表见第 4 节，前端禁止裸抛后端 message。

## 1. 调用方式：wx.cloud.callContainer 配置

### 1.1 环境 ID（envId）

| 环境 | envId 来源 | 维护人 |
|---|---|---|
| 开发 | 微信公众平台 → 云开发 → 环境列表，复制"环境 ID" | 后端 |
| 生产 | 同上，单独建一个生产环境 | 后端 |

envId 不写死在代码里，统一放 `frontend/src/config/env.ts`：

```ts
export const ENV = {
  dev:  { cloudEnvId: 'fortune-dev-1xxxxxxx',  service: 'fortune-backend' },
  prod: { cloudEnvId: 'fortune-prod-1xxxxxxx', service: 'fortune-backend' }
}
export const CURRENT = process.env.NODE_ENV === 'production' ? ENV.prod : ENV.dev
```

### 1.2 X-WX-SERVICE 写法

- 值 = 云托管"服务名"，固定 `fortune-backend`。
- 必须和小程序后台「云托管 → 服务列表」名称完全一致，区分大小写。

### 1.3 path 规则

- 后端所有路由统一前缀 `/api/`，见 02 文档第 4 节。
- callContainer 的 `path` 必须以 `/` 开头，**不要拼 envId、不要拼域名**。
- query 参数直接拼在 path 里：`/api/fortune/history?page=1&pageSize=10`。

### 1.4 请求封装样板

```ts
import { CURRENT } from '@/config/env'

interface ApiResp<T> { code: number; message: string; data: T; traceId: string }

export function call<T = any>(opt: {
  path: string
  method?: 'GET' | 'POST' | 'DELETE'
  data?: any
  header?: Record<string, string>
}): Promise<ApiResp<T>> {
  return new Promise((resolve, reject) => {
    wx.cloud.callContainer({
      config: { env: CURRENT.cloudEnvId },
      path: opt.path,
      method: opt.method || 'GET',
      header: {
        'X-WX-SERVICE': CURRENT.service,
        'content-type': 'application/json',
        ...opt.header
      },
      data: opt.data,
      success: (res: any) => resolve(res.data as ApiResp<T>),
      fail: reject
    })
  })
}
```

注意：
1. `content-type` 用小写。
2. 不要在前端塞 `X-WX-OPENID`，云托管侧会被覆盖或拒绝。
3. 返回的 `res.data` 才是后端真正的 `ApiResponse<T>`。

## 2. USE_MOCK 开关说明

### 2.1 配置位置

```ts
export const USE_MOCK = false  // true = 走前端静态 JSON, false = 走 callContainer 真接口
```

### 2.2 切换效果矩阵

| 配置 | 数据来源 | 网络请求 | 适用场景 |
|---|---|---|---|
| `USE_MOCK = true` | `frontend/src/mock/*.json` | 不发出 | 没后端、UI 自测、设计稿评审 |
| `USE_MOCK = false`（H5） | `uni.request` 直连 `http://localhost:8080` | 发出 | 本地联调 |
| `USE_MOCK = false`（mp-weixin） | `wx.cloud.callContainer` | 发出 | 真机调试 / 提审 / 上线 |

### 2.3 api 层分流写法

```ts
import { USE_MOCK } from '@/config/env'
import mockCalc from '@/mock/fortune_calc.json'
import { call } from './request'
import { httpLocal } from './request.local'

export function calcFortune(req) {
  if (USE_MOCK) return Promise.resolve({ code: 0, data: mockCalc, message: 'ok', traceId: 'mock' })
  // #ifdef H5
  return httpLocal.post('/api/fortune/calc', req)
  // #endif
  // #ifdef MP-WEIXIN
  return call({ path: '/api/fortune/calc', method: 'POST', data: req })
  // #endif
}
```

注意：
1. mock JSON 字段必须和 02 文档接口契约**完全一致**。
2. 上线前必须把 `USE_MOCK` 改回 `false`，建议加 CI 脚本扫描。
3. 提审包禁止包含 `frontend/src/mock/`。

## 3. 联调环境矩阵

| 维度 | 本地开发（H5 调试） | 微信开发者工具调试 | 生产 |
|---|---|---|---|
| 前端启动 | `pnpm dev:h5` | `pnpm dev:mp-weixin` + 微信开发者工具导入 `dist/dev/mp-weixin` | 体验版/正式版 |
| 后端运行方式 | `mvn spring-boot:run` | 云托管开发环境（dev 镜像） | 云托管生产环境 |
| 调用方式 | `uni.request` 直连 `localhost:8080` | `wx.cloud.callContainer` | 同左 |
| openid 来源 | 前端 Header 手填 mock_openid | 云托管自动注入 | 云托管自动注入 |
| CORS | 后端放行 `*`（dev profile） | 不需要 | 不需要 |

**前端本地 http 封装（仅 H5 用）：**

```ts
const BASE = 'http://localhost:8080'
export const httpLocal = {
  post: (path: string, data: any) => uniRequest('POST', path, data),
  get:  (path: string)            => uniRequest('GET',  path),
  del:  (path: string)            => uniRequest('DELETE', path)
}
function uniRequest(method, path, data?) {
  return new Promise((resolve, reject) => {
    uni.request({
      url: BASE + path, method, data,
      header: {
        'content-type': 'application/json',
        'X-WX-OPENID': 'mock_openid_local_dev'   // 本地伪造
      },
      success: r => resolve(r.data), fail: reject
    })
  })
}
```

## 4. 错误码对照表 + 前端 toast 文案

| code | 含义 | 触发示例 | 前端表现 | toast 文案 |
|---|---|---|---|---|
| 0 | 成功 | - | 按业务处理 | - |
| 1001 | 参数缺失 | calc 缺 birthday | 留在当前页 | "信息没填完，再检查一下～" |
| 1002 | 参数格式错 | gender = "X" | 留在当前页 | "格式好像不太对哦" |
| 1003 | 参数超范围 | 生日选了未来 | 高亮该字段 | "生日不能晚于今天哦" |
| 2001 | 未登录 | token 失效 | 静默重新 login，重试 1 次 | "请重新打开小程序" |
| 2002 | 记录不存在 | delete 一条已不存在的记录 | 列表本地移除 | "这条已经不见啦" |
| 2003 | 未解锁 | 直接访问 deepContent | 引导看激励视频 | "看完小视频就能解锁～" |
| 4001 | 限流 | calc 1 分钟超过 10 次 | 按钮禁用 30s | "测得太快啦，歇一会儿再来" |
| 5000 | 服务端异常 | 后端 NPE | 兜底文案 + 重试按钮 | "网络开小差，等会再试试" |
| 网络错误 | 请求 fail | 断网、超时 | 兜底文案 | "网络不太好，检查一下信号" |

**前端统一处理样板：**

```ts
const TOAST = {
  1001: '信息没填完，再检查一下～',
  1002: '格式好像不太对哦',
  1003: '生日不能晚于今天哦',
  2002: '这条已经不见啦',
  2003: '看完小视频就能解锁～',
  4001: '测得太快啦，歇一会儿再来',
  5000: '网络开小差，等会再试试'
}
export function handleResp(resp) {
  if (resp.code === 0) return resp.data
  if (resp.code === 2001) { throw new Error('NEED_RELOGIN') }
  uni.showToast({ title: TOAST[resp.code] || '出错啦', icon: 'none' })
  throw new Error(`BIZ_${resp.code}`)
}
```

## 5. openid 处理约定

- 本期后端 `AuthService` 是 mock，但 `/api/auth/login` 接口必须保留。
- **生产链路**：所有 `/api/*` 请求经过云托管，云托管自动在 Header 注入 `X-WX-OPENID`、`X-WX-APPID`，前端零代码。
- **本地 H5 链路**：手动在 `uni.request` 的 Header 里塞 `X-WX-OPENID: mock_openid_local_dev`，后端 `OpenIdResolver` 兼容。
- **后端读取**：

```java
String openid = request.getHeader("X-WX-OPENID");
if (openid == null) openid = "mock_openid_default"; // 兜底，仅 dev profile 允许
```

> 线上风险：上线前必须确认 `prod` profile 下 fallback 关闭；mock_openid 不能进生产日志看板；后续接 DB 后所有"删除/解锁"接口必须做归属校验。

## 6. 联调 8 个用例

### 用例 1：login → calc → history 列表能看到记录
- 步骤：冷启动 → 录入页填生日 1995-08-15、性别 M → 提交。
- 接口：`POST /api/auth/login` → `POST /api/fortune/calc` → `GET /api/fortune/history?page=1&pageSize=10`。
- 前端表现：跳转结果页，显示 score、宜忌、四大维度概要；切到历史 Tab 看到记录。

### 用例 2：同一用户当天再次 calc，score 一致
- 步骤：结果页"再测一次" → 不改资料 → 提交。
- 前端表现：score、level、lucky 与上次完全一致；历史不重复增加。

### 用例 3：delete 单条 → 列表少一条
- 步骤：历史列表，左滑某条 → 删除 → 二次确认。
- 前端表现：动画淡出该条，total - 1，toast"已删除"。

### 用例 4：delete 全部 → 列表空
- 步骤：历史页右上角"清空" → 二次确认。
- 前端表现：列表立即变空状态插画。

### 用例 5：unlock → 返回 deepContent；再 calc 同记录 unlocked=true
- 步骤：结果页"查看详情" → 看完激励视频 → 回调成功。
- 前端表现：折叠区展开，展示 monthTrend / yearKeyword / advice / compatibleZodiac。

### 用例 6：短时间 11 次 calc，第 11 次 4001
- 前端表现：toast"测得太快啦"，按钮禁用 30s 倒计时。

### 用例 7：birthday 未来 → 1003
- 前端表现：定位到生日字段红框，toast"生日不能晚于今天哦"。

### 用例 8：gender X → 1002
- 前端表现：toast"格式好像不太对哦"。

## 7. 常见联调问题排查

### 7.1 接口 404
1. 看 path 是否带 `/api/` 前缀（最常见漏前缀）。
2. 后端启动日志是否打印对应路由 `Mapped "{[/api/fortune/calc]"`。
3. 云托管控制台 → 服务详情 → 访问设置 → 路径是否配了 `/api/*`。
4. 镜像版本是否最新（云托管「版本管理」看 100% 流量指向）。

### 7.2 跨域报错（仅 H5 调试出现）
- 后端 dev profile 的 `CorsConfig` 必须放行 `*`。
- 微信开发者工具/真机不会出现，只在 H5 调试出现。
- 临时方案：`pnpm dev:h5 --proxy http://localhost:8080`。

### 7.3 callContainer 路径未在小程序后台配置白名单
**症状**：`errMsg: callContainer:fail path not in service config`。
- 微信公众平台 → 云开发 → 云托管 → 服务 `fortune-backend` → 「路径配置」 → 新增 `/api/*`。

### 7.4 其他高频坑

| 现象 | 真因 | 处理 |
|---|---|---|
| 真机能跑、开发者工具不能跑 | 没勾"不校验合法域名" | 勾上 |
| openid 一直是空 | 用了 `wx.request` 而不是 `wx.cloud.callContainer` | 改回 callContainer |
| 时间差 8 小时 | 后端 Docker 没设 TZ | Dockerfile 加 `ENV TZ=Asia/Shanghai` |
| 偶发 5000 | 内存 Mock 文案 JSON 加载失败 | 看后端日志 `MockDataService` 启动 log |
| 同一天 score 跳变 | 服务端时区不一致 / openid 拿到了 mock_default | 见上两条 |
