> 文档版本：V1.0 ｜ 日期：2026-04-30
> 适用：产品 / 数据 / 运营 / 前端
> 配套基线：`01-PRD.md` 第 3 / 7 节

## 0. 结论先看

- 埋点统一通过 `utils/tracker.ts` 上报，**事件名格式 `页面_动作`**，全小写下划线。
- 广告 4 类：Banner / 原生模板 / 激励视频 / 插屏，按 PRD 第 7 节策略落点。
- 数据看板 4 个核心指标：DAU / 人均测算次数 / 激励视频完成率 / ARPU 估算。
- 本期不接第三方数据平台，**复用微信小程序后台「数据分析」+「流量主」+ 自定义 `wx.reportEvent`**。

## 1. 埋点规范

### 1.1 命名规则

- 事件名：`{page}_{action}`，如 `home_view`、`result_unlock_click`。
- 字段：`event`、`ts`（毫秒时间戳）、`openid`（云托管自动注入侧拿）、`extra`（业务字段对象）。
- 上报方式：开发期 `console.log`，上线后调 `wx.reportEvent`。

### 1.2 上报封装

```ts
// frontend/src/utils/tracker.ts
export function track(event: string, extra: Record<string, any> = {}) {
  const payload = { event, ts: Date.now(), ...extra }
  // #ifdef MP-WEIXIN
  try { wx.reportEvent(event, extra) } catch (_) {}
  // #endif
  if (process.env.NODE_ENV !== 'production') console.log('[track]', payload)
}
```

### 1.3 注意事项

- 不上报手机号、生日、姓名等隐私字段（合规红线）。
- `extra` 字段不超过 10 个，单值长度 ≤ 64。
- 高频事件做本地节流（如滚动）。

## 3. 广告位列表

### 3.1 广告位总表

| 序号 | 位置 | 类型 | unitId 占位 | 触发时机 | 频次/上限 | 关联曝光埋点 | 关联完成埋点 |
|---|---|---|---|---|---|---|---|
| AD-01 | 首页底部 | Banner（原生模板） | `adunit-home-banner-xxxx` | 页面加载 | 常驻 | `home_banner_show` | `home_banner_click` |
| AD-02 | 结果页摘要下 | 原生模板 | `adunit-result-native-xxxx` | 进入页面 1s 后 | 每次 1 个 | `result_native_show` | - |
| AD-03 | 结果页"查看详情" | 激励视频 | `adunit-result-reward-xxxx` | 用户主动点击 | 单页最多 4 次 | `result_reward_show` | `result_reward_complete/close/fail` |
| AD-04 | 结果页离开 | 插屏 | `adunit-result-interstitial-xxxx` | 返回 / 切 Tab | 每用户每日最多 1 次 | `result_interstitial_show` | - |
| AD-05 | 历史列表 | 信息流 | `adunit-history-native-xxxx` | 列表中每 5 条插入 1 个 | 列表常驻 | `history_native_show` | - |
| AD-06 | 个人中心底部 | Banner | `adunit-mine-banner-xxxx` | 加载 | 常驻 | `mine_banner_show` | - |

```ts
  RESULT_INTERSTITIAL:'adunit-result-interstitial-xxxx',
  HISTORY_NATIVE:     'adunit-history-native-xxxx',
  MINE_BANNER:        'adunit-mine-banner-xxxx'

```

### 3.3 频控实现要点

| 广告位 | 频控规则 | 实现位置 | 存储 |
|---|---|---|---|
| AD-03 激励视频 | 单页 ≤ 4 次 | 结果页内 ref 计数 | 内存（页面级） |
| AD-04 插屏 | 单用户每日 ≤ 1 次 | 路由钩子 | `uni.setStorageSync('lastInterstitialDate', 'yyyy-MM-dd')` |
| 启动 5s 内 | 不出插屏 | 路由钩子 | 比较 `App.onLaunch` 时间戳 |

### 3.4 注意事项

- 激励视频不与签到 / 积分 / 虚拟货币挂钩。
- 所有广告位必须有"广告"标识（小程序基础库自带）。
- 插屏不能挡住「关闭」按钮，不能在打开 1s 内出现。
- 18 岁以下用户场景下，结果页感情维度屏蔽。
- 任一广告位 7 日填充率 < 50% 时运营介入。

## 4. 数据看板预期指标

### 4.1 核心 4 指标

| 指标 | 定义 | 数据来源 | 目标值（MVP 上线 1 个月） |
|---|---|---|---|
| **DAU** | 当日 `home_view` 去重 openid 数 | 小程序后台「数据分析」 | 5,000 |
| **人均测算次数** | 当日 `calc_submit_success` 总数 / DAU | 自定义事件聚合 | 1.3 |
| **激励视频完成率** | `result_reward_complete` / `result_reward_show` | 流量主后台 + 自定义事件交叉 | ≥ 60% |
| **ARPU 估算** | 当日广告总收入 / DAU | 流量主后台 | ¥ 0.10 ~ ¥ 0.30 |

### 4.2 漏斗指标

| 漏斗节点 | 事件 | 目标转化率 |
|---|---|---|
| 启动 → 点 CTA | `home_view` → `home_cta_click` | ≥ 40% |
| 点 CTA → 提交 | `calc_submit_click` → `calc_submit_success` | ≥ 90% |
| 进结果页 → 点详情 | `result_view` → `result_dimension_click` | ≥ 35% |
| 点详情 → 看完视频 | `result_reward_show` → `result_reward_complete` | ≥ 60% |
| 看完视频 → 解锁成功 | `result_reward_complete` → `result_unlock_success` | ≥ 95% |

### 4.3 留存指标

| 指标 | 定义 | 目标 |
|---|---|---|
| 次日留存 | 今天新增明天回来 | ≥ 25% |
| 7 日留存 | 今天新增第 7 天回来 | ≥ 12% |
| 连续签到 ≥ 3 日占比 | `home_signin_click` streak ≥ 3 占比 | ≥ 15% |

### 4.4 ARPU 估算口径

```
日广告收入 = Σ(各广告位 eCPM × 曝光数 / 1000)

参考 eCPM（业内中位）:
  Banner / 原生模板 ≈ ¥ 3 ~ 8
  插屏              ≈ ¥ 8 ~ 20
  激励视频          ≈ ¥ 30 ~ 80

举例 DAU = 5000 时单日预估:
  AD-01 Banner    曝光 5000 × ¥5  / 1000 = ¥ 25
  AD-02 原生      曝光 4000 × ¥6  / 1000 = ¥ 24
  AD-03 激励视频  曝光 1500 × ¥50 / 1000 = ¥ 75
  AD-04 插屏      曝光  500 × ¥15 / 1000 = ¥ 7.5
  AD-05 信息流    曝光 2000 × ¥6  / 1000 = ¥ 12
  AD-06 Banner    曝光 1500 × ¥5  / 1000 = ¥ 7.5
  合计 ≈ ¥ 151 → ARPU ≈ ¥ 0.03

> 上述为保守估算，实际以流量主后台为准。激励视频 eCPM 是收入大头，运营优先盯它。
```

### 4.5 看板搭建建议

- 第 1 阶段（MVP 上线 0~2 周）：直接用「微信小程序后台 → 数据分析 + 自定义分析」，不投入额外开发。
- 第 2 阶段（DAU > 1 万）：考虑接入腾讯有数 / 自建 BI。
- 关键日报字段：DAU、新增、留存、人均测算、激励视频完成率、广告收入、ARPU。
